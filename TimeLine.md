# 武侠客栈经营游戏开发时间表

## 总体时间线概览

| 阶段 | 时间 | 核心目标 |
| :--- | :--- | :--- |
| **阶段一** | 第1-4周 | 核心原型：基础经营循环 + 武侠氛围 |
| **阶段二** | 第5-8周 | 家具系统深度集成 + 经济系统完善 |
| **阶段三** | 第9-12周 | 内容填充 + 江湖事件系统 |
| **阶段四** | 第13-16周 | 打磨、优化、测试与发布准备 |

---

## 阶段一：核心原型与基础框架（第1-4周）

**目标**：做出一个“可玩的最小版本”——客人能入住、付钱、离开，客栈能盈利。

### 第1周：项目搭建与基础系统

- [ ] 创建Unity项目，设置文件夹结构（Scripts, Scenes, Prefabs, Data等）
- [ ] 导入基础资源（免费3D模型、UI素材）
- [ ] 实现**摄像机控制**（俯视视角、缩放、旋转）
- [ ] 搭建基础场景：地面、客栈边界、一个简单的房间
- [ ] 实现**客人AI基础**：生成客人 → 移动到前台 → 分配到房间 → 退房离开

**里程碑**：一个客人能完整走完入住→退房流程。

---

### 第2周：经济系统与UI

- [ ] 实现**经济系统**：银两变量、收入/支出计算
- [ ] 实现**基础UI**：当前银两、客栈等级、客人满意度
- [ ] 实现**动态定价**：根据入住率自动调整房价
- [ ] 添加**客栈基础属性**：房间数、清洁度、声誉值
- [ ] 实现**每日结算**：自动扣租金/工资，显示每日盈亏

**里程碑**：客栈能自动运营，玩家能看到每日收入和支出。

---

### 第3周：武侠氛围基础

- [ ] 引入**客人类型原型**：侠客（要高武力值房间）、文人（要安静）
- [ ] 添加**说书人NPC**：在大堂讲江湖故事，提升客栈声望
- [ ] 实现**基础任务系统**：如“接待三位侠客获得额外奖励”
- [ ] 设计**江湖背景设定文档**：门派、势力、关键人物

**里程碑**：客栈有明显的“武侠感”，而不只是一个普通酒店。

---

### 第4周：家具系统原型（与阶段一并行）

- [ ] 实现**基础家具放置**：从UI选择家具 → 预览 → 放置
- [ ] 实现**网格系统**（参考之前提供的`GridSystem.cs`）
- [ ] 实现**简单碰撞检测**：家具不能重叠、不能超出客栈边界
- [ ] 实现**家具数据管理**：`FurnitureData` ScriptableObject创建10种基础家具

**里程碑**：玩家能放置/移动/删除家具，并保存布局。

---

## 阶段二：家具系统深度集成与经营深度（第5-8周）

**目标**：让家具**影响游戏玩法**（而只是装饰），同时完善经济平衡。

### 第5周：家具与游戏系统联动

- [ ] 实现**家具功能化**：
  - 椅子+桌子 → 客人用餐，产生额外收入
  - 床 → 提升睡眠质量，增加满意度
  - 兵器架 → 吸引侠客，增加武力值
- [ ] 实现**员工系统基础**：雇佣店小二，店小二需要“休息椅”
- [ ] 实现**家具影响满意度**：客人会评价“房间太简陋”

**里程碑**：放置餐桌后，客人能坐下吃饭并付钱。

---

### 第6周：寻路系统与动态NavMesh

- [ ] 集成**Unity NavMesh**，实现NPC绕开家具行走
- [ ] 实现**动态NavMesh烘焙**：放置/移动家具后自动更新寻路
- [ ] 优化**家具碰撞检测**：支持不规则形状家具
- [ ] 实现**多层客栈**（如果计划有二楼）：区分不同楼层的网格

**里程碑**：放置家具后，客人会绕开行走，不会卡住。

---

### 第7周：经济平衡与数值设计

- [ ] 设计**完整的家具价格表**（20+种家具）
- [ ] 实现**客栈升级系统**：升级后解锁新家具、扩大面积
- [ ] 实现**贷款系统**：玩家可以向钱庄借钱
- [ ] 进行**初步数值平衡**：确保前期不破产，后期不无聊

**里程碑**：从0开始，玩家能在1小时内发展到中级客栈。

---

### 第8周：家具系统优化

- [ ] 实现**家具旋转时的碰撞实时检测**（已提供代码）
- [ ] 实现**撤销/重做功能**（Ctrl+Z / Ctrl+Y）
- [ ] 实现**家具布局保存/加载多槽位**（可保存多套方案）
- [ ] 添加**家具商店UI**：分类显示、价格、预览

**里程碑**：家具系统稳定无Bug，体验流畅。

---

## 阶段三：内容填充与江湖事件（第9-12周）

**目标**：让游戏有“讲不完的故事”，增加重玩价值。

### 第9周：丰富客人AI与需求

- [ ] 增加5种以上客人类型（镖师、乞丐、采花贼、朝廷密探）
- [ ] 实现**客人需求系统**：气泡显示“饿”、“想切磋”、“要热水”
- [ ] 实现**江湖恩怨**：仇家相遇会打架，玩家可选择劝架或围观
- [ ] 实现**随机事件基础框架**

**里程碑**：客栈变成一个“有故事发生的地方”。

---

### 第10周：事件系统深度开发

- [ ] 实现**20+随机事件**：
  - 踢馆事件 → 武力对决
  - 神秘商人 → 出售稀有家具
  - 官府查案 → 选择包庇或举报
- [ ] 实现**道德/声望系统**：正派/邪派选择影响后续事件
- [ ] 实现**季节与节日**：春节客栈装饰、中秋猜灯谜

**里程碑**：玩家每次开新档，事件体验都不完全相同。

---

### 第11周：任务与成长线

- [ ] 实现**主线任务**（如“从破客栈到武林第一”）
- [ ] 实现**支线任务**（如“帮助受伤的侠客送药”）
- [ ] 实现**成就系统**：解锁特殊家具、称号
- [ ] 添加**图鉴系统**：记录遇到的客人、解锁的家具

**里程碑**：玩家有清晰的长期目标。

---

### 第12周：音效与视觉反馈

- [ ] 添加**背景音乐**（古风、轻松）
- [ ] 添加**音效**：银两声、开门声、客人欢呼/抱怨
- [ ] 实现**视觉特效**：家具高亮、满意/不满意气泡
- [ ] 实现**简单的角色动画**：走路、坐下、吃饭

**里程碑**：游戏视听完整，有沉浸感。

---

## 阶段四：打磨、优化、测试与发布（第13-16周）

**目标**：确保游戏稳定、流畅、有趣。

### 第13周：性能优化

- [ ] **优化寻路**：减少NavMesh烘焙频率，使用对象池
- [ ] **优化家具渲染**：LOD（多细节层次）、剔除不可见物体
- [ ] **优化内存**：资源加载/卸载策略
- [ ] **测试极限情况**：100个客人+50个家具同时运行

**里程碑**：低配电脑也能流畅运行。

---

### 第14周：Bug修复与平衡调整

- [ ] 内部测试，收集Bug清单
- [ ] 修复**家具穿透、客人卡墙**等严重Bug
- [ ] 根据测试调整**经济平衡**（避免太快/太慢）
- [ ] 优化**新手引导**：前30分钟不迷茫

**里程碑**：游戏无明显Bug，新手能轻松上手。

---

### 第15周：用户测试与反馈

- [ ] 邀请5-10名玩家试玩
- [ ] 收集反馈：难度曲线、UI易用性、趣味性
- [ ] 根据反馈进行**最后调整**
- [ ] 添加**设置选项**：音量、画质、控制方式

**里程碑**：玩家愿意主动推荐给朋友。

---

### 第16周：发布准备

- [ ] 制作**宣传视频/截图**（Steam/itch.io）
- [ ] 撰写**商店描述、更新日志**
- [ ] 进行**最终打包测试**（Windows/Mac/Linux）
- [ ] 上传到平台，**发布！**

**里程碑**：游戏成功发布！

---

## 灵活调整建议

- **如果时间紧张**：砍掉“阶段三”的部分事件，优先保证核心循环稳定。
- **如果想做移动端**：在第13周额外增加“触控适配”和“离线收益”开发。
- **如果想加入多人模式**：需要额外8-12周，建议作为DLC或后续更新。
