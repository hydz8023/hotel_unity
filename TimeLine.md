# 武侠客栈经营游戏 — 多版本开发时间表

> 依据 [doc/idear.md](doc/idear.md) 策划案制定。采用**多版本、小步交付**：每个版本结束都有可运行、可演示的 build，避免 16 周内铺全所有系统。
>
> **当前工程基线（已完成）**：摄像机控制、`GridSystem` / `FurniturePlacer` / `PlacementPreview` / `LayoutSaver` 网格摆放与 JSON 存档（见 [AGENTS.md](AGENTS.md)）。

---

## 版本总览

| 版本 | 建议工期 | 定位 | 玩家可体验到的核心 |
| :--- | :--- | :--- | :--- |
| **v0.1** | —（已完成） | 摆放原型 | 俯视相机、网格放置/移动/删除、布局存档 |
| **v0.2** | 3–4 周 | 经营原型 | 客人入住→退房、银两日结、玩家手动接待收银 |
| **v0.3** | 3–4 周 | 武侠氛围版 | 侠客/文人、门口小乞丐风闻、1 家被动对手、基础任务 |
| **v0.4** | 4–5 周 | 人岗经营版 | 基础四岗、家具功能化、NavMesh、菜谱与客房标签 |
| **v0.5** | 4–5 周 | 江湖经营版 | 势力好感、扩展员工、风闻前瞻、三档难度、事件框架 |
| **v1.0** | 4–6 周 | 首发完整版 | 夜间竞争、包打听、主线、打磨发布 |
| **v1.x** | 持续迭代 | 内容与扩展 | 更多事件、剧本、楼层、子门派等 |

**累计至 v1.0 首发**：约 **22–28 周**（按单人/小团队估算）。原「16 周全做完」改为 **v1.0 目标**，中间版本均可单独试玩或对外小规模测试。

## v0.1 — 摆放原型（已完成）

**目标**：验证网格布置与存档链路，为后续「布局影响经营」打地基。

### 交付清单

- [x] Unity 项目与目录结构（`Assets/script/`、`Assets/Script/`）
- [x] **摄像机控制**：`WASD` 平移、`Q/E` 旋转、滚轮缩放
- [x] **网格系统**：`GridSystem` 吸附、占用检测、Gizmos 坐标

- [x] **购买预览**：`PlacementPreview`

- [ ] 10 种基础家具资源与 ScriptableObject 配置（内容可随 v0.2 补齐）

**里程碑**：玩家能布置客栈并保存/重载布局。

## v0.2 — 经营原型（第 1–4 周）

**目标**：在已有场景上跑通**最小经营闭环**，暂不追求武侠深度；玩家默认**身兼四岗**（手动接待、送餐、做菜、收银）。

> 对应 idear §3.1 分钟循环 + §4.1 经济，不含员工 AI。

### 第 1 周：场景与客人骨架

- [ ] 客栈场景：地面、边界、前台、通铺/地字房（先用固定房间，不必接标签）
- [ ] **客人 AI 基础**：生成 → 前台 → 分配房间 → 停留 → 退房
- [ ] 简单 NavMesh 或点对点移动（家具少时可先静态路径）
- [ ] 客人数据：`GuestData` ScriptableObject 或配置表雏形

**周里程碑**：一名客人完整走完入住→退房。

### 第 2 周：经济与 UI

- [ ] **银两**、房费收入、固定支出（租金）
- [ ] **每日结算**面板：收入/支出/盈亏
- [ ] HUD：银两、满意度（可先简化为单一数值）
- [ ] **动态定价**雏形（按入住率调价，可先手动开关）
- [ ] 客栈属性：房间数、清洁度、声誉值（声誉可先仅显示）

**周里程碑**：客栈能「自动跑一天」并看到日结报表。

### 第 3 周：手动服务链

- [ ] 客人需求：**住宿 + 饿了要吃饭**（点击接单→点击厨房→点击上菜→点击收银）
- [ ] 无员工时的 **手动 fallback** UI（对应 idear §4.5.3）
- [ ] 3–5 道**固定菜谱**（一键出餐，暂无食材库存）
- [ ] 桌椅、床、灶台与满意度/收入的简单挂钩

**周里程碑**：玩家亲手完成一单「住宿+餐饮」并正确入账。

### 第 4 周：整合与存档

- [ ] **经营存档**（银两、日数、客栈等级）与布局存档分离或合并方案
- [ ] Bug 修复：客人卡死、重复结账、日结边界
- [ ] 内部试玩：30 分钟内能理解目标且不破产（数值粗调）
- [ ] 补齐 v0.1 家具 SO 与基础 Prefab

**版本里程碑（v0.2）**：**能赚钱的客栈模拟器**——有布置、有客人、有日结，武侠皮尚未上。

## v0.3 — 武侠氛围版（第 5–8 周）

**目标**：贴上武侠标签，引入**风闻引导**与**轻度竞争**，仍保持系统精简。

> 对应 idear §4.3 客人类型原型、§4.10 风闻（指南类）、§4.11 对手（仅白日客流）。

### 第 5 周：江湖客人

- [ ] **侠客 / 文人**两种客人：不同房型偏好（先用简单规则，完整标签留 v0.4）
- [ ] 需求气泡：安静 / 兵器架等（不匹配则满意度下降）
- [ ] 客人带 `factionId` 字段（阶段一只做显示，无数值）

**周里程碑**：侠客与文人会挑不同房间，差评有文案反馈。

### 第 6 周：风闻与小乞丐

- [ ] **门口小乞丐 NPC**（点击交互）
- [ ] **风闻簿** UI：指南类风闻（人岗、操作提示）
- [ ] 5–10 条固定指南文案，对应当前已实现功能
- [ ] 未读角标与已读标记

**周里程碑**：新玩家通过乞儿能理解「没人时要自己端菜」等规则。

### 第 7 周：被动对手与基础任务

- [ ] **1 家对手客栈**（如悦来客栈）：仅 **客流占比** 争夺，无夜间行动
- [ ] 声誉影响本镇客流分配（简化公式）
- [ ] **3–5 个基础任务**：如「接待 3 位侠客」「日盈利达 X」
- [ ] 任务奖励：银两、声望、单件家具

**周里程碑**：能看到「今日客流被对街抢去一成」的反馈。

### 第 8 周：氛围整合

- [ ] 简单江湖背景音乐 / 关键 SFX（银两、开门，可选）
- [ ] 客人与乞丐对话润色
- [ ] v0.2–v0.3 联合测试与数值粗调
- [ ] 打 **v0.3 标签包**，写简短更新说明

**版本里程碑（v0.3）**：**像武侠客栈了**——有江湖客、有风闻、有对手压力，适合小范围试玩。

## 裁剪原则（全版本通用）

1. **先闭环、后皮肤**：没有日结的经营，不加夜间竞争。
2. **先手动、后 AI**：四岗一律先能做手动 fallback，再雇员工。
3. **先白日、后夜间**：对手先抢客流，再偷菜造谣。
4. **先数值条、后叙事网**：势力先 4 条好感，再子门派与多结局。
5. **每版本可删**：当週延期时，砍「扩展」保「闭环」——见下表。

| 若延期，优先砍 | 不可砍 |
| :--- | :--- |
| 音效/额外客人/成就 | 本版本里程碑闭环 |
|  segundo 对手、部分事件 | 当前版本核心系统 |
| 铁人模式、第三结局 | 日结、存档、引导 |
