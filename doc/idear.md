## 1. 游戏概述

### 1.1 游戏名称（暂定）
- 待定

### 1.2 游戏类型
- 模拟经营 + 武侠题材 + 网格化空间布置
- 俯视视角、单机、轻度策略

### 1.3 核心卖点
- **经营客栈**：接待江湖客、赚取银两、升级扩张
- **自由布置**：网格化摆放家具，布局直接影响经营效率与客人满意度
- **江湖故事**：随机事件、恩怨纠葛、正邪抉择，让客栈成为「有故事的地方」
- **风闻情报**：门口小乞丐兜售消息与指南，助玩家未雨绸缪
- **同业暗战**：与镇上对手客栈夜偷菜谱、说书造谣，争夺客流与声望
- **菜谱与分区**：研发招牌菜、给客房贴标签，布局与餐饮共同决定客流
- **势力周旋**：正派、邪派、朝廷、商帮好感影响客人、事件与长期结局

### 1.4 目标平台
- PC（Windows / Mac / Linux）
- 可选扩展：移动端触控、离线收益（见 TimeLine 灵活调整建议）

### 1.5 目标玩家
- 喜欢模拟经营、布置类玩法的休闲玩家
- 对武侠/古风题材有兴趣的玩家

## 2. 世界观与背景设定

### 2.1 时代与舞台
- 架空武侠江湖，玩家经营一家从破败到名震武林的客栈

### 2.2 江湖势力（见 4.14）

- **四大势力**：正派、邪派、朝廷、商帮（可再细分子门派）
- 势力好感影响客人类型、独家任务、随机事件与主线分支
- 关键 NPC 作为各势力代表，与客栈声望、风闻联动

### 2.3 客栈定位
- 玩家角色：**客栈东家**（老板），开局往往身兼数职；可逐步雇佣员工接管岗位
- 长期目标：武林第一客栈（主线任务锚点）

### 2.4 氛围基调
- 古风、轻松、略带江湖恩怨与抉择
- 门口乞儿、说书人、节日、江湖传闻强化武侠感
- 镇上不止一家客栈，暗地里的菜谱之争与口舌之争贯穿中后期

## 3. 核心玩法循环

### 3.1 分钟级循环（单次操作）
```
客人到来 → 前台接待/分配房间（掌柜或玩家手动）
→ 满足需求：厨子出菜、跑堂送餐，或玩家亲自操办
→ 账房结账或玩家手动收银 → 收取费用 → 满意度与声望变化
```
- **人岗原则**：某岗位无员工时，对应环节由**玩家手动完成**；员工到岗后自动接管，玩家可专注布置与决策

### 3.2 日循环
- 客人入住与退房
- 动态定价随入住率调整
- 每日结算：收入、租金、工资、盈亏报表

### 3.3 周/月级循环
- 购买与摆放家具，优化布局；为客房设置**分区与标签**（见 4.13）
- 研发与调整**菜单菜谱**，管理食材库存（见 4.12）
- 雇佣员工、客栈升级、扩建面积
- 触发随机事件、完成主线/支线任务；经营**势力好感**（见 4.14）
- 查看**风闻**、应对**竞争对手**动作（夜袭菜谱、谣言攻防）

### 3.4 长期循环
- 声望与道德（正邪）影响后续事件与客人构成
- 成就、图鉴、特殊家具与称号解锁
- 多档布局方案保存与切换

## 4. 系统设计

### 4.1 经济系统

| 要素 | 说明 |
| :--- | :--- |
| 货币 | 银两 |
| 收入 | 房费、餐饮（菜谱售价）、特殊服务、任务奖励 |
| 支出 | 租金、员工工资、家具购置、贷款利息 |
| 动态定价 | 根据入住率自动调整房价 |
| 贷款 | 可向钱庄借钱，支撑前期扩张，普通钱庄利率低但额度底，黑心商人九出十三归 |
| 升级 | 客栈等级解锁新家具、扩大可布置区域 |

**数值目标**（阶段二里程碑）：从 0 开始，玩家约 1 小时内可发展到中级客栈；前期不易破产，后期有成长空间。

### 4.2 客栈属性

| 属性 | 作用 |
| :--- | :--- |
| 房间数 | 决定同时接待上限 |
| 清洁度 | 影响满意度与客人评价 |
| 声誉值 | 影响客人类型、事件触发、定价上限 |
| 客栈等级 | 解锁内容与面积、员工编制上限 |
| 满意度 | 综合反馈，关联收入与口碑 |
| 安全度 | 护院、事件后果；闹事与踢馆概率（扩展员工加成） |
| 餐饮/服务效率 | 厨子、跑堂岗位与武功技能、厨房动线、菜谱熟练度综合体现 |
| 菜品口碑 | 招牌菜、菜谱品质；影响餐饮收入与客流构成（见 4.12） |

### 4.3 客人系统

#### 4.3.1 客人类型（分阶段引入）

| 阶段 | 类型示例 | 核心需求 |
| :--- | :--- | :--- |
| 原型 | 侠客、文人 | 兵器房 / 静修房等房间标签（见 4.13） |

#### 4.3.2 客人 AI 流程
1. 生成 → 移动至前台
2. 掌柜分配房间（匹配**房间标签**与客人需求；见 4.13）
3. 入住期间产生需求（饿 → 点菜单菜品、切磋、热水等）
4. 退房离开，结算费用与评价；部分客人携带**势力身份**影响好感（见 4.14）

#### 4.3.3 江湖恩怨
- 仇家同场可能发生冲突
- 玩家选择：劝架 / 围观 / 其他（待设计分支后果）

### 4.4 家具与布置系统

> 技术实现见 AGENTS.md：网格左下角为逻辑原点 `(0,0)`，摆放须同步占用格与 JSON 存档。

#### 4.4.1 核心交互
- 从商店/UI 选择家具 → 预览 → 网格吸附放置
- 拖拽移动、旋转、删除
- 碰撞与边界检测：不可重叠、不可越界
- 布局持久化：`furnitureId` + `PlacedFurniture`，支持多槽位方案

#### 4.4.2 家具分类与功能（由装饰走向玩法）

| 家具 | 功能影响 |
| :--- | :--- |
| 床 | 提升睡眠质量 → 满意度 |
| 桌椅 | 客人用餐 → 额外收入 |
| 兵器架 | 吸引侠客 → 武力相关加成 |
| 休息椅 | 员工（跑堂等）恢复体力，维持服务效率 |
| 装饰类 | 房间档次、客人评价（「太简陋」等） |
| 灶台 / 橱柜 | 解锁菜谱烹饪、食材存储上限（见 4.12） |
| 兵器架、古琴、屏风等 | 赋予或强化客房**标签**（见 4.13） |

#### 4.4.3 家具数据规模
- 阶段一：约 10 种基础家具（`FurnitureData` ScriptableObject）
- 阶段二：20+ 种，含完整价格表与商店分类 UI

#### 4.4.4 布置体验增强（阶段二）
- 旋转时实时碰撞检测
- 撤销/重做（Ctrl+Z / Ctrl+Y）
- 多套布局方案保存与加载

#### 4.4.5 与寻路联动（阶段二）
- Unity NavMesh，家具放置后动态烘焙
- NPC 绕开家具行走，避免卡墙

### 4.5 员工系统

员工是连接「家具布局」与「日常经营」的关键：有岗有人则自动运转，缺岗则压到玩家身上。

#### 4.5.1 设计原则

| 原则 | 说明 |
| :--- | :--- |
| **人岗对应** | 每项基础经营功能绑定固定岗位；该岗位无人时，只能玩家手动操作 |
| **基础 vs 扩展** | 基础员工维持客栈运转；扩展员工提供额外属性加成与氛围 |
| **武侠技能** | 员工带江湖武功/技艺词条，影响岗位效率与特殊互动 |
| **待遇与忠诚** | 工资、休息设施不足会导致效率下降或离职（难度可开关） |

#### 4.5.2 基础员工（维持运转）

四类岗位为客栈**最低运转配置**，缺一则对应流程卡顿或完全依赖玩家。

| 岗位 | 职责 | 无员工时（玩家手动） | 依赖家具/区域 |
| :--- | :--- | :--- | :--- |
| **掌柜** | 前台接待、分配房间、处理简单纠纷 | 点击客人办理入住/退房；高峰时易排队超时 | 柜台、大堂 |
| **跑堂** | 传菜、送水、收拾餐桌、响应客房呼叫 | 点击订单逐单配送；多桌同时需求时压力大 | 桌椅、客房通道；**休息椅**恢复体力 |
| **厨子** | 接单烹饪、控制出菜速度 | 点击厨房手动做菜；客流高时出菜瓶颈明显 | 灶台、厨房区域 |
| **账房** | 收银结账、日终对账、防止漏收 | 每笔消费需玩家确认收款；日结需玩家手动核对 | 账房桌、柜台 |

**岗位协作链**（餐饮示例）：
```
客人点餐 → 掌柜/系统记单 → 厨子烹饪 → 跑堂送餐 → 账房结账
         ↑ 任一环节缺人且玩家未介入 → 订单停滞、客人耐心下降
```

#### 4.5.3 玩家手动模式

- 开局默认**四岗皆空**，玩家身兼东家与所有杂务，教学成本低、操作密度高
- 雇佣员工后，该岗位 AI 接管；玩家仍可**临时插手**（如紧急送一单菜），但不替代长期雇人收益
- UI 提示：缺岗岗位在 HUD 显示警示（如「无跑堂 · 3 桌待送」），引导玩家去招工

#### 4.5.4 扩展员工（属性加成）

基础四岗齐备后，可雇佣**扩展岗位**，不替代基础功能，主要为客栈叠加**可量化属性**与事件互动。

| 岗位 | 解锁阶段 | 客栈属性加成（示例） | 说明 |
| :--- | :--- | :--- | :--- |
| **说书人** | 阶段三 | 声望增长 +、大堂氛围、吸引文人客 | 需大堂茶座；讲书期间触发江湖传闻事件 |
| **护院** | 阶段三 | 安全度 +、压制闹事概率 | 踢馆/打架事件更易控场 |
| **采买** | 扩展 | 食材成本 −、库存上限 + | 减少随机涨价冲击 |
| **琴师 / 杂役等** | 扩展 | 满意度、清洁度等 | 按内容量迭代 |
| **包打听**（原门口乞儿） | 阶段三任务解锁 | 风闻自动推送、竞争情报、预警 | 见 4.10；雇后常驻门口或大堂 |

扩展员工与 4.2 客栈属性的映射示例：

| 客栈属性 | 主要来源 |
| :--- | :--- |
| 声誉值 | 说书人、高满意度、事件 |
| 安全度 | 护院、兵器架布局 |
| 餐饮效率 | 厨子、灶台等级 |
| 服务速度 | 跑堂、轻功技能、动线布局 |
| 经营稳健 | 账房、珠算技能、漏收率 |
| 情报优势 | 包打听、风闻系统；预知事件与对手动向 |

#### 4.5.5 武功与技艺技能

员工除岗位基础能力外，拥有 **1–2 个武功/技艺词条**，招聘时随机或按简历选择，可培养（扩展）。

| 岗位 | 技能示例 | 游戏效果 |
| :--- | :--- | :--- |
| **跑堂** | **轻功**高 | 移动速度 +、跨障碍送菜、同时接单数 + |
| **跑堂** | 铁布衫 | 劝架时不易被打伤，闹事损失 − |
| **厨子** | **刀法**精 | 出菜速度 +、批量订单处理 |
| **厨子** | 内功 | 夜间班次疲劳衰减慢 |
| **掌柜** | **口才** | 谈价成功率 +、挑剔客人耐心 + |
| **掌柜** | 威慑 | 小混混闹事概率 − |
| **账房** | **珠算** | 结账速度 +、漏收/错账率 − |
| **账房** | 过目不忘 | 日结报表自动标出异常支出 |
| **说书人** | 内力深厚 | 讲书声望加成更高、吸引江湖名流 |
| **护院** | 拳掌 / 兵器 | 驱离闹事者成功率、踢馆应战选项 |
| **包打听** | 轻功、耳功 | 夜间盯梢竞争对手、风闻刷新质量 + |

**技能等级**：初窥门径 / 小有名气 / 一代高手（影响数值幅度与稀有招聘池）

**与布置联动**：跑堂轻功再高，厨房与饭厅之间堵死过道仍会降效——技能放大布局收益，不替代合理摆放。

#### 4.5.6 雇佣与管理

| 要素 | 说明 |
| :--- | :--- |
| 招聘 | 客栈等级、声望达标后开放；人才市场随机刷新带技能词条的候选人 |
| 工资 | 计入每日固定支出；技能越好要价越高 |
| 编制上限 | 随客栈等级提升；同岗可雇多人（如 2 跑堂）分摊高峰 |
| 休息 | 无休息椅或连续夜班 → 效率 − 或请假 |
| 离职 | 长期欠薪、客栈声望过低时触发（高难度更明显） |

#### 4.5.7 分阶段落地

| 阶段 | 员工内容 |
| :--- | :--- |
| 阶段一 | 玩家全手动；可选仅实现 **跑堂** 原型验证 AI 送餐 |
| 阶段二 | 四类基础员工齐备 + 缺岗手动 fallback + 休息椅联动 |
| 阶段三 | 说书人、护院等扩展员工 + 武功技能词条 + 招聘 UI |
| 阶段四 | 技能培养、同岗多人排班、员工相关成就 |

### 4.9 难度与模式设计

难度不应只靠「敌人更强」一类单一旋钮，而应拆成若干**可感知的经营压力维度**，让玩家在开档或设置里组合选择。本游戏建议从以下方式设定难度，并可分阶段落地。

#### 4.9.1 预设难度档（开档三选一）

适合首版快速上手，底层用一张「难度系数表」驱动各系统倍率。

| 档位 | 定位 | 典型差异（相对普通） |
| :--- | :--- | :--- |
| **闲云野鹤** | 轻松布置、少破产 | 初始银两 +50%；日支出 −20%；客人耐心 +30%；负面事件 −40% |
| **江湖掌柜** | 默认体验，平衡成长 | 基准数值 |
| **龙门客栈** | 高压经营、抉择更重 | 初始银两 −30%；房租/工资 +25%；挑剔客人 +50%；事件惩罚加重 |

可选第四档 **自定义**：解锁各维度滑块（见 4.9.8），供老玩家微调。

#### 4.9.2 经济压力

直接作用于「会不会破产、扩张快不快」，与 4.1 经济系统、贷款规则联动。

| 调节项 | 高难度表现 | 低难度表现 |
| :--- | :--- | :--- |
| 初始资金 | 少，需精打细算 | 充裕，可试错摆放 |
| 固定支出（租金、工资） | 涨幅快、基数高 | 涨幅缓、有新手减免期 |
| 收入倍率 | 房费/餐饮加成上限低 | 同布局下收入更高 |
| 贷款 | 额度低、利率高（黑心商人更早出现） | 普通钱庄额度高、利息低 |
| 破产线 | 负债达上限即 Game Over | 允许多次「宽限」，或仅扣声望 |
| 物价波动 | 家具/食材随机涨价 | 价格稳定 |

#### 4.9.3 客人强度与客流

作用于分钟级、日循环的操作密度与容错。

| 调节项 | 高难度表现 | 低难度表现 |
| :--- | :--- | :--- |
| 客流密度 | 高峰时段同时多桌、多房 | 客人错峰到达 |
| 客人耐心 | 等待过久易不满、退房 | 等待容忍度高 |
| 需求复杂度 | 同时出现「饿 + 切磋 + 热水」等复合需求 | 单次需求简单、间隔长 |
| 挑剔客人占比 | 侠客/密探等高要求类型比例高 | 普通旅人为主 |
| 付费意愿 | 同满意度下小费/房费偏低 | 更愿意加价、打赏 |
| 差评惩罚 | 一次不满显著拉低声誉 | 声誉恢复快、单次影响小 |

#### 4.9.4 客栈运营负担

不增敌人，而是让「同样布局」更难维持。

| 调节项 | 高难度表现 | 低难度表现 |
| :--- | :--- | :--- |
| 清洁度衰减 | 掉得快，需频繁打理 | 衰减慢或自动缓慢恢复 |
| 房间/家具磨损 | 需维修支出（扩展项） | 无磨损或维修便宜 |
| 员工效率 | 同样人数服务更少桌/房 | 跑堂移速与接单量更高 |
| 员工离职 | 待遇不足易离职 | 忠诚度更高 |
| 同时接待上限 | 初期房间少、升级贵 | 起步房间多 |

#### 4.9.5 布置与空间约束

与 4.4 家具系统、网格布局强相关，适合「爱动脑」型难度。

| 调节项 | 高难度表现 | 低难度表现 |
| :--- | :--- | :--- |
| 可布置面积 | 小网格、升级才扩地 | 起步面积大 |
| 家具价格 | 贵，试错成本高 | 折扣或首购优惠 |
| 功能家具依赖 | 必须桌椅+床+兵器架才接得住某类客 | 缺家具时惩罚轻 |
| 寻路/拥堵 | 布局不佳时客人更易堵路、超时 | 拥堵惩罚弱或提示强 |

#### 4.9.6 江湖事件与叙事压力

与 4.7 事件系统、正邪声望联动。

| 调节项 | 高难度表现 | 低难度表现 |
| :--- | :--- | :--- |
| 事件触发频率 | 更勤，常打断日常经营 | 前期事件少，教学向 |
| 负面/抉择事件占比 | 踢馆、查案、恩怨多 | 奖励型、节日型事件多 |
| 选项后果 | 选错损失大、声望难挽回 | 后果温和、可补救 |
| 仇家冲突 | 同场打架概率高、劝架失败代价大 | 冲突少或自动化解 |
| 同业竞争 | 对手夜间行动勤、造谣伤害大 | 对手被动，竞争可关闭 |
| 主线时限 | 章节目标带倒计时 | 无硬性时限 |

#### 4.9.7 开局剧本（场景难度）

在预设档位之上叠加**固定起点**，改变「第一小时体验」而非全局倍率。

| 剧本示例 | 难点 |
| :--- | :--- |
| 破败路边店 | 资金少、家具残、客流不稳 |
| 边境黑店 | 官府事件多、邪派客人多，正邪抉择频繁 |
| 山门脚小店 | 侠客需求高、易起切磋事件 |
| 繁华镇店 | 租金高、竞争激烈，需精细定价 |

同一剧本可与任意预设档位组合，例如「边境黑店 + 龙门客栈」。

#### 4.9.8 自定义维度（滑块 / 开关）

首版可只做 3 档预设；阶段三后增加「高级难度」面板，建议暴露项：

- 初始银两、支出倍率、收入倍率
- 客人数量与耐心
- 事件频率与恶性事件占比
- 是否开启破产 Game Over
- 是否开启员工离职 / 家具磨损（扩展）
- 游戏内时间流速（见 4.9.9）

各滑块应**实时预览一句话说明**（如「预计第 3 日房租压力：偏高」），避免数值黑箱。

#### 4.9.9 时间与节奏

| 调节项 | 高难度表现 | 低难度表现 |
| :--- | :--- | :--- |
| 一日时长（现实时间） | 短，操作仓促 | 长，可仔细布置 |
| 暂停/倍速 | 仅暂停，无 3 倍速 | 可慢速、可加速结算 |
| 季节周期 | 冬闲夏忙差异大 | 全年客流较平稳 |
| 离线收益（若做移动端） | 无或打折 | 满额离线收益 |

#### 4.9.10 失败与存档规则

难度也体现在「能不能重来、输得起多少」。

| 模式 | 说明 |
| :--- | :--- |
| **标准** | 破产可读档上一次每日结算 |
| **铁人** | 单存档、破产即删档，成就加权 |
| **沙盒** | 无破产、银两下限兜底，偏布置与剧情 |
| **挑战周目** | 通关后解锁 modifiers（如「全局清洁度 −30%」） |

#### 4.9.11 难度与成长 / 成就的挂钩

- 高难度通关解锁：特殊称号、家具皮肤、图鉴条目
- 部分成就仅在高难度或铁人模式可拿，避免逼肝普通档
- 主线「武林第一客栈」可设**难度无关**的叙事终点，但高难追加隐藏结局（如邪派线、朝廷册封）

#### 4.9.12 推荐落地顺序

| 阶段 | 建议实现 |
| :--- | :--- |
| 阶段一 | 仅「江湖掌柜」单档；经济数值按低破产率调 |
| 阶段二 | 三档预设 + 初始资金/支出/收入系数表 |
| 阶段三 | 开局剧本 2–3 个、事件频率挂钩难度 |
| 阶段四 | 自定义滑块、铁人/沙盒、用户测试校准各档通关率 |

**设计原则**：高难度应来自「更多 simultaneous 决策 + 更紧的资源」，而非单纯「数值恶心」；每档难度都需保证前 30 分钟目标清晰（见 5.3）。

### 4.10 风闻系统

风闻是游戏的**情报与软引导层**：用江湖口吻传递操作提示、近期变数与对手动向，让玩家有时间提前布置、囤货或雇人。

#### 4.10.1 入口：门口小乞丐

- 客栈大门外固定出现**小乞丐 NPC**，开局即可见、可点击
- 交互形式：短对话 + **风闻簿**面板（可回看近几条消息）
- 定位：不抢戏、不替玩家做决定，只提供「听说了什么」

#### 4.10.2 风闻类型

| 类型 | 内容示例 | 设计目的 |
| :--- | :--- | :--- |
| **游戏指南** | 「掌柜爷，没跑堂时可得您亲自端菜呐」「账房不在，钱得您亲手点」 | 隐性教学，对应 4.5 人岗机制 |
| **经营前瞻** | 「后日中秋，文人客会多」「菜价要涨，趁早囤粮」 | 让玩家提前调整定价、采买、家具 |
| **江湖预告** | 「镇上来了踢馆的」「官府要查黑店」 | 对接 4.7 随机事件，预留准备时间 |
| **竞争情报** | 「对街悦来客栈在招厨子」「听说他们新菜式不错」 | 对接 4.11 竞争对手 |
| **闲话轶闻** | 门派恩怨、侠客行踪 | 氛围与任务线索 |

**剧透边界**：前瞻信息用模糊表述（「三两日内」「恐有变故」），不精确到秒；已触发的事件不再作为「预告」。

#### 4.10.3 刷新规则

| 阶段 | 规则 |
| :--- | :--- |
| 初期 | 点击小乞丐时刷新 1 条；指南类权重高 |
| 中期 | 每日清晨自动推送 1 条到风闻簿；点击可追问 1 条补充 |
| 雇包打听后 | 自动推送条数 +、竞争情报权重 +；偶发「紧急风闻」弹窗 |

风闻条目可标记 **已读 / 已筹备**，避免信息淹没（UI 勾选「已备料」等）。

#### 4.10.4 与经营的联动

- 玩家根据风闻提前：**进货、调价、加雇临时跑堂、布置节日装饰、加强夜间守夜**
- 忽视风闻不阻断流程，但相关事件发生时惩罚或机会窗口更紧（如未囤粮则节日利润率低）
- 成就：「听风就是雨」——连续 N 次根据风闻完成筹备

#### 4.10.5 入职：包打听

小乞丐可通过 **支线任务链「乞儿入行」** 转为正式员工 **包打听**（4.5.4 扩展岗）。

| 环节 | 内容 |
| :--- | :--- |
| 触发 | 客栈声望达阈值 + 与小乞丐好感够 |
| 任务示例 | 赠衣赠食、帮其打听一次对手虚实、击退欺负乞儿的混混 |
| 入职效果 | 风闻自动推送；竞争情报更具体；可指派「盯梢某客栈」夜间行动 |
| 技能倾向 | 耳功、轻功（与 4.5.5 一致） |

入职后门口不再显示流浪乞儿形象，改为员工制服；风闻簿入口移至 HUD 或包打听头像。

### 4.11 竞争对手系统

玩家客栈并非独霸一方；**镇上另有 1–3 家对手客栈**（名称、风格、掌柜性格各异），形成长期客流与声望争夺。

#### 4.11.1 对手画像

| 对手类型 | 特点 | 常见手段 |
| :--- | :--- | :--- |
| **悦来客栈** | 平价走量 | 压价、抢普通旅人 |
| **醉仙楼** | 菜品出名 | 推新菜、挖厨子 |
| **黑店** | 邪派背景 | 造谣、夜间偷窃、恶意闹事 |

对手也有**声誉、菜品库、员工编制**，随游戏进度成长，避免早期碾压玩家。

#### 4.11.2 竞争维度（白日）

| 维度 | 说明 |
| :--- | :--- |
| 客流分配 | 同镇总客流固定池，声望、定价、特色菜影响占比 |
| 定价战 | 对手降价时风闻预警；玩家跟价损利润、不跟则丢客 |
| 挖角 | 对手高薪诱走己方厨子/说书人；提高工资或任务挽留 |
| 菜品口碑 | 独有菜谱带来客流加成；菜谱被窃则优势削弱 |

#### 4.11.3 夜间行动（核心玩法）

每日打烊后进入**夜间阶段**（可简化为事件选择界面，不必做完整夜间场景），玩家与对手可互施手段：

**① 偷菜谱**

| 要素 | 说明 |
| :--- | :--- |
| 己方发起 | 选派 **厨子**（潜行）或 **跑堂**（轻功高者更佳）潜入对手后厨 |
| 成功率 | 受员工技能、对手护院、己方风闻情报质量影响 |
| 成功 | 习得对方菜式副本（收入加成或解锁烹饪）；对手次日声望 − |
| 失败 | 员工受伤请假、声誉 −、遭对手报复或官府介入 |
| 防守 | 雇**护院**、厨房上锁（家具）、包打听预警可降低被窃概率 |

**② 说书造谣**

| 要素 | 说明 |
| :--- | :--- |
| 己方发起 | 需雇佣**说书人**；选择谣言类型（食品安全、黑店、闹鬼等） |
| 效果 | 对手声誉 −、客流短期流向己方；文人客可能追问真相 |
| 风险 | 谣言被识破 → 己方声誉大跌、触发查案类事件 |
| 对手同样可用 | 玩家需通过风闻、客人抱怨察觉，选择澄清、反造谣或公关 |

**③ 其他夜间手段（扩展）**

- 撒钉拦客、贿赂说书人、伪造差评纸条等——按内容量迭代，共用「夜间行动点」上限

#### 4.11.4 行动点与节奏

- 每夜玩家有 **1–2 点夜间行动**（升级账房/包打听可增加）
- 未使用的行动点不累积，避免囤点一波碾压
- 对手 AI 按性格与局势选择偷菜/造谣/休整；高难度对手更主动

#### 4.11.5 与风闻、任务、主线的关系

- **风闻**是竞争的主要预警来源（「对街昨夜有人翻墙」）
- **主线**可含「挤垮黑店」「三家联名办武林宴」等章节
- **包打听**入职后解锁对手情报面板（声誉条、估计客流、昨夜动向）

#### 4.11.6 难度与开关

| 设置 | 说明 |
| :--- | :--- |
| 闲云野鹤 | 对手被动，夜间行动少 |
| 江湖掌柜 | 默认竞争强度 |
| 龙门客栈 | 对手主动偷菜/造谣，失败惩罚重 |
| 沙盒模式 | 可关闭竞争，专注布置与剧情 |

#### 4.11.7 分阶段落地

| 阶段 | 内容 |
| :--- | :--- |
| 阶段一 | 门口小乞丐 + 指南类风闻 + 1 家被动对手（仅白日客流争夺，无夜间） |
| 阶段二 | 经营前瞻风闻；对手数据面板雏形 |
| 阶段三 | 夜间偷菜谱 / 造谣完整链路 + 包打听任务链 + 2–3 家性格对手 |
| 阶段四 | AI 平衡、竞争成就、用户测试「是否太肝/太阴」 |

### 4.12 菜谱体系

餐饮是客栈第二收入支柱，也是武侠氛围（名菜、秘方、偷师）的载体。菜谱体系与厨子、食材、风闻、竞争对手（4.11）深度耦合。

#### 4.12.1 核心概念

| 概念 | 说明 |
| :--- | :--- |
| **菜谱** | 可学习、升级的菜品数据；含食材消耗、烹饪时间、售价、客群倾向 |
| **食材** | 库存物资；采买、节日涨价、风闻预警；不足则无法出菜 |
| **菜单** | 玩家配置当日在售菜品（上限 3–5 道），决定客流类型与厨房负荷 |
| **招牌菜** | 限设 1–2 道，额外口碑加成，竞品最想偷的目标 |

#### 4.12.2 菜谱来源

| 途径 | 说明 |
| :--- | :--- |
| **初始菜谱** | 开局 3–5 道基础家常菜 |
| **研发** | 厨子消耗食材试菜，概率解锁新菜谱；刀法高者成功率 + |
| **任务奖励** | 支线、势力任务赠送独家菜 |
| **偷学** | 夜间行动从对手客栈窃取（4.11）；获得「副本菜谱」，熟练度从 0 起 |
| **神秘商人** | 稀有食材 + 菜谱残页 |

#### 4.12.3 菜谱属性

| 属性 | 作用 |
| :--- | :--- |
| 成本 / 售价 | 利润与账房结算 |
| 烹饪时间 | 厨子负荷；高峰易积压订单 |
| 客群标签 | 如「侠客喜」「文人喜」「平价」；影响点单率 |
| 时令 | 部分菜仅特定季节满分，非时令满意度降 |
| 熟练度 | 厨子反复制作升级；速度 +、废品率 − |
| 品质档 | 普通 / 精品 / 绝品（家具、食材、技能共同决定） |

#### 4.12.4 经营流程

```
采买食材 → 配置菜单/招牌菜 → 客人点餐 → 厨子烹饪 → 跑堂上菜 → 账房结账
              ↑ 无厨子则玩家手动烹饪          ↑ 缺菜则客人不满或改点
```

- **风闻联动**：「菜价要涨」「对街新菜出名」促使玩家调价、研发或偷学
- **竞争联动**：招牌菜口碑进入全镇菜品排行，影响客流分配（4.11.2）

#### 4.12.5 与员工、家具的联动

| 要素 | 联动 |
| :--- | :--- |
| 厨子 | 刀法影响出菜速度与批量处理；内功影响夜班疲劳 |
| 采买 | 降低食材单价、提高库存上限 |
| 灶台等级 | 解锁高阶菜谱、缩短烹饪时间 |
| 厨房动线 | 厨子与跑堂路径影响「从下单到上菜」总耗时 |

#### 4.12.6 分阶段落地

| 阶段 | 内容 |
| :--- | :--- |
| 阶段一 | 3–5 道固定菜、无库存（简化为一键出餐） |
| 阶段二 | 食材库存、菜单配置、熟练度；与餐饮收入挂钩 |
| 阶段三 | 研发、招牌菜、时令菜、偷学菜谱与竞争排行 |
| 阶段四 | 菜谱全表平衡、绝品触发条件、相关成就 |

### 4.13 客房分区与房间标签

客栈空间不仅是网格，还应划分为**功能区**与**带标签的客房**，供掌柜自动或玩家手动分房，匹配不同江湖客需求。

#### 4.13.1 空间分区

随客栈等级解锁不同区域（可在同一网格内用区域标记实现）：

| 分区 | 功能 |
| :--- | :--- |
| **大堂** | 接待、用餐、说书、擂台（扩展） |
| **厨房** | 烹饪、仓储 |
| **客房区** | 住宿核心；可再分楼上/楼下 |
| **后院** | 晾晒、堆货、护院巡逻；扩展练功场 |
| **账房 / 柜台** | 结账、日结 |

分区影响 NavMesh、员工巡逻范围与事件触发位置（如踢馆多发生在大堂）。

#### 4.13.2 房型与档次

| 房型 | 说明 | 定价 |
| :--- | :--- | :--- |
| **通铺** | 多客合住，满意度要求低 | 低 |
| **地字房** | 标准单间 | 中 |
| **天字房** | 宽敞、家具齐全 | 高 |
| **小院 / 独栋** | 后期解锁，VIP 与长期包房 | 最高 |

#### 4.13.3 房间标签（核心玩法）

每个可售客房可拥有 **1–2 个标签**，由**家具组合 + 玩家挂牌**共同决定：

| 标签 | 达成条件（示例） | 适宜客人 | 不匹配后果 |
| :--- | :--- | :--- | :--- |
| **静修** | 屏风、远离大堂、无嘈杂家具 | 文人、道士 | 满意度 −、差评「太吵」 |
| **兵器** | 兵器架、练武地毯 | 侠客、镖师 | 文人不安 |
| **雅室** | 古琴、书画、高档床榻 | 文人、商贾 | — |
| **暖阁** | 炭盆、厚被褥 | 畏寒客、冬季全体 | 冬季满意度降 |
| **通衢** | 默认，无特殊家具 | 普通旅人 | — |

- 玩家在家具摆好后，对房间**挂牌**确认标签（如「天字一号 · 静修」）
- 有掌柜时，分房 AI **优先匹配**标签与客人需求；无掌柜时玩家手动选房

#### 4.13.4 与布置、客人的联动

- 同一房型因家具不同而标签不同，鼓励**同尺寸房间的差异化布置**
- 客人需求气泡可显示「要静修房」「要能搁剑的屋」
- 竞争对手不抢菜谱时，也可打「客房档次」战（需风闻与口碑系统配合）

#### 4.13.5 数据与持久化

- 房间标签随 **InnLayoutData** 存档：房间 ID + 标签列表 + 房型
- 移动/拆除关键家具时，标签失效或变灰，UI 提示重新挂牌

#### 4.13.6 分阶段落地

| 阶段 | 内容 |
| :--- | :--- |
| 阶段一 | 通铺 + 地字房，无标签（仅房型差价） |
| 阶段二 | 静修 / 兵器 / 雅室三标签 + 掌柜自动分房 |
| 阶段三 | 天字房、小院、暖阁等季节标签；图鉴记录客房方案 |
| 阶段四 | 多楼层分区、标签与事件联动（密探偏爱隐蔽房等） |

### 4.14 江湖势力好感

在全局「声誉值」之外，增加与**四大势力**的独立好感，让玩家长期经营「立场」与「人脉」，而不只是堆一个声望数字。

#### 4.14.1 四大势力

| 势力 | 代表人群 | 好感高时的收益 | 好感低时的风险 |
| :--- | :--- | :--- | :--- |
| **正派** | 侠客、名门弟子 | 正义事件奖励、踢馆支援 | 少来侠客、邪派闹事增多 |
| **邪派** | 采花贼、黑道、杀手 | 黑市渠道、高额违禁委托 | 正派踢馆、官府查案 |
| **朝廷** | 官兵、密探、税吏 | 免税额、官方庇护 | 查封、高额罚款 |
| **商帮** | 商贾、镖局关联商人 | 采买折扣、稳定大宗客流 | 食材涨价、商路断绝事件 |

势力下可再挂子门派（少林、武当等）作为后期扩展，首版四势力即可。

#### 4.14.2 好感度规则

| 规则 | 说明 |
| :--- | :--- |
| 数值范围 | 各势力独立：−100（敌对）～ +100（敬重） |
| 初始值 | 默认中立 0；开局剧本可偏移（如边境黑店邪派 +20） |
| 变化来源 | 事件抉择、接待对应客人、完成任务、说书造谣内容、拒绝/包庇某人 |
| 对冲 | 讨好邪派常损正派；朝廷与邪派常对立——**不可四面讨好** |
| 与道德线 | 正邪声望作为「总倾向」；势力好感作为「分势力明细」 |

#### 4.14.3 好感带来的玩法

| 层级 | 效果 |
| :--- | :--- |
| **冷淡（低于 −30）** | 该势力客人稀少；负面事件权重 + |
| **友善（高于 +30）** | 该势力客人占比 +；解锁势力商店/任务 |
| **敬重（高于 +60）** | 独家菜谱、员工推荐、势力援护（踢馆帮腔） |
| **敌对（低于 −60）** | 主动闹事、暗杀、官府举报等高危事件 |

#### 4.14.4 与现有系统的联动

| 系统 | 联动方式 |
| :--- | :--- |
| **客人** | 每位客人带 `factionId`；接待满意度影响小幅好感 |
| **事件 4.7** | 选项标注势力影响预览（「正派 +10，邪派 −15」） |
| **风闻 4.10** | 「少林弟子近日多」「官府要查黑店」 |
| **竞争 4.11** | 对手可投靠势力，拉偏镇上的势力平衡 |
| **菜谱 4.12** | 势力独家菜需好感达标解锁 |
| **任务 4.6** | 主线章节分势力线；支线为各派办事 |
| **难度** | 龙门客栈好感对冲更剧烈，抉择更难 |

#### 4.14.5 UI 与反馈

- **势力关系面板**：四象限或四条声望条，显示当前等级称谓（如「朝廷：通缉中」「商帮：贵客」）
- 事件弹窗显示**好感变化预览**，避免玩家误触不可逆后果
- 图鉴记录各势力关键人物与历史抉择

#### 4.14.6 分阶段落地

| 阶段 | 内容 |
| :--- | :--- |
| 阶段一 | 不显示数值；客人仅带势力标签做氛围 |
| 阶段二 | 四势力好感条 + 简单事件加减 |
| 阶段三 | 势力任务、独家菜、对冲与主线分支 |
| 阶段四 | 子门派、隐藏结局（朝廷册封 / 邪派盟主等） |

## 5. UI / UX 设计要点

### 5.1 常驻 HUD
- 当前银两、客栈等级、客人满意度（及可选：声誉、清洁度）
- **风闻**入口（信封/乞儿图标）；未读条数角标
- 竞争态势简条（阶段三+）：本镇客流占比或对手声誉对比
- 势力好感简条（阶段二+）：四势力图标与等级色（可选收纳进面板）

### 5.2 功能界面
- 家具商店：分类、价格、预览
- 每日结算面板：盈亏明细
- 任务/成就/图鉴入口
- 布局方案管理（多槽位）
- **员工管理**：岗位编制、招聘（技能词条预览）、工资、缺岗警示
- **风闻簿**：分类浏览（指南 / 前瞻 / 竞争）、已读与筹备标记
- **夜间行动**（打烊后）：偷菜谱、造谣、守夜等选项与成功率预览
- **对手情报**（包打听入职后）：各家客栈概况与昨夜动向
- **菜单/菜谱**：在售菜品、食材库存、招牌菜设置、研发入口
- **客房管理**：房型、分区视图、房间标签挂牌与匹配状态
- **势力关系**：四势力好感、称谓、势力任务入口
- **新游戏**：预设难度档、开局剧本选择；高级难度展开自定义滑块（阶段三+）

### 5.3 反馈与引导
- 可放置/不可放置材质高亮
- 客人需求气泡、满意/不满意反馈
- **门口小乞丐**承担前期软引导；风闻簿可回看机制说明
- 新手引导：前 30 分钟目标清晰、不迷茫

### 5.4 相机控制（已实现）
- `WASD` 平移、`Q/E` 水平旋转、滚轮缩放
- 旋转绕当前视角中心公转

## 7. 技术约束与实现映射

> 与代码仓库现状对齐，策划需求落地时需遵守以下约定。

| 策划概念 | 技术对应 |
| :--- | :--- |
| 家具定义 | `FurnitureData`（ScriptableObject） |
| 家具库 | `FurnitureDatabase` |
| 运行时实例 | `FurnitureItem` |
| 布局存档 | `InnLayoutData` + `PlacedFurniture` |
| 存取入口 | `LayoutSaver`（JSON，`persistentDataPath`） |
| 网格与占用 | `GridSystem` |
| 摆放流程 | `FurniturePlacer`、`PlacementPreview` |
| 相机 | `CameraController` |

### 7.1 摆放不变式（策划需知）
- 任何改变家具位置/旋转的操作，必须同时更新网格占用与存档
- 取消/撤销须正确回滚占用状态
- 公共序列化字段改名需迁移方案

### 7.2 性能目标（阶段四）
- 低配可流畅运行；压力场景参考：约 100 客人 + 50 家具
- NavMesh 烘焙频率、对象池、LOD 等优化项
